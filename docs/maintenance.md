# 维护和升级流程

`agent-evals-cn` 的目标是持续升级，而不是一次性写完。

## 每次升级前

1. 收集真实失败案例。
2. 把失败案例写成 eval case。
3. 跑当前版本，记录 baseline。
4. 明确本次只解决哪些失败模式。

## 每次升级中

1. 小步修改 `SKILL.md`。
2. 避免同时改目标、判分标准和实现规则。
3. 如果新增规则会增加上下文负担，考虑移到 `docs/` 或 `examples/`。

## 每次升级后

1. 重跑已有 eval cases。
2. 记录新增通过项和仍失败项。
3. 更新 README 或示例。
4. 写清楚已知限制。

## 版本号建议

- `0.x`：探索期，规则和模板可以频繁变化。
- `1.0`：至少有 20 个稳定 eval cases，且覆盖 4 类以上 agent 工作流。
- patch：修正文档、模板或小规则。
- minor：新增评测类别或评分维度。
- major：改变核心评测方法或输出格式。
