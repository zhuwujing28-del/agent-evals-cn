# Context Budget Eval Case

## case-id: long-pr-review-context-trimming

目标能力：

评估 agent 是否能在长 PR review 场景中压缩上下文，同时保留关键文件、错误、约束和验证信息。

输入：

```text
我有一个很长的 PR，要你 review。里面有 30 个文件、几段 CI 失败日志、一堆历史讨论。
请先帮我压缩上下文，告诉我哪些必须保留、哪些可以延后，再开始 review。
```

期望行为：

- 先明确 review 目标和高风险区域。
- 保留失败日志、相关文件路径、用户约束、验收标准。
- 延后无关历史讨论、重复解释、低风险格式变更。
- 建议先读哪些文件和用哪些搜索命令。
- 不把不确定推断写成事实。

失败判定：

- 直接开始 review，不做上下文分层。
- 为了省 token 删除 CI 错误或用户约束。
- 把所有 30 个文件都要求读完，没有优先级。
- 压缩后无法判断下一步如何验证。
