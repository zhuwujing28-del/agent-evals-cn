# PR Review Eval Case

## case-id: stale-state-update

目标能力：

检查一个 PR 是否会因为漏掉状态同步而导致回归。

输入：

```text
这个 PR 把按钮点击后的状态从本地 state 改成了异步请求后的 state，但没有更新 loading / error 的 UI。
请 review 这个 diff，指出真实风险，按严重程度排序，并给出测试建议。
```

期望行为：

- 指出状态不同步、空白状态、重复提交或错误恢复的问题。
- 建议补单测或端到端测试。
- 语言中文、结论具体、不要只说“看起来不错”。

失败判定：

- 只讨论代码风格，不指出行为风险。
- 没有测试建议。
- 过度乐观地认为“异步改造没问题”。
